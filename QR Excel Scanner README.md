# 📷 QR Excel Scanner

> 條碼 / QR Code 掃描工具，即時將掃描結果填入 Excel 或 Word 游標位置，無需手動輸入。

---

## 專案簡介

**QR Excel Scanner** 是一款專為倉儲、製造、驗貨等需要大量條碼掃描場景所設計的桌面工具。透過 USB 攝影機即時掃描 QR Code 或 Code128 條碼，並透過 Windows COM 介面自動將結果填入目前 Excel 或 Word 游標所在的儲存格，完全省去手動鍵入的繁瑣步驟。

程式採用 Tkinter 圖形介面，支援完整模式與**簡易模式**（可浮於 Excel 上層、不遮擋視線），是一套真正可在實際作業環境使用的生產力工具。

---

## 功能特色

### 🔍 條碼掃描
- 支援 **QR Code**、**Code128** 等主流格式（pyzbar 引擎）
- 背景執行緒掃描，不影響畫面更新的流暢度（~33 FPS）
- 可設定**同碼冷卻時間**（預設 5 秒），避免重複掃入同一條碼
- 可設定每碼最大掃描次數上限

### 📊 直接填入 Office
- 透過 **Windows COM / pywin32** 讀取 Excel / Word 游標位置
- 掃描成功後自動填入**目前游標所在儲存格**，無需點選任何視窗
- 支援自動下移游標（可設定延遲時間），批次掃描更順暢
- 數字開頭為 `0` 的條碼自動套用文字格式，防止 Excel 自動截斷前導零
- 支援**撤銷（Undo）**最近一筆填入

### 🔄 進階填入模式
- **MAC / SN 篩選模式**：僅擷取 MAC 地址或序號格式的條碼
- **MAC 後 N 碼擷取**：可設定只擷取 MAC 後 2 / 4 / 6 / 8 / 12 碼
- **交替填入模式**：MAC → SN → MAC → SN 自動交替填入相鄰欄位

### 🎥 攝影機控制
- 支援多路攝影機切換（Camera Index 0、1、2...）
- 支援解析度選擇：SD / HD / FHD / 2K
- **數位放大**（1× ~ 4×），滑鼠拖拽可調整放大中心
- **手動 / 自動對焦**切換，並學習並記憶「黃金焦距」
- **閒置鎖焦**：閒置超過 30 秒後自動鎖定焦距，防止 AF 漂移失焦
- **自動曝光補償**：環境光不足時自動調整，確保辨識率

### 🖥️ 介面設計
- **完整模式**：攝影機預覽 + 完整設定面板
- **簡易模式**：縮小至攝影機視窗，可置頂浮於 Excel 上層，不干擾操作
- **Toast 通知**：每次成功填入後彈出小視窗確認，支援釘選或自動消失
- 所有設定（攝影機、對焦、曝光、填入模式）自動儲存至 JSON，下次啟動自動還原
- 支援**置頂顯示**（Always on Top），方便多視窗環境操作

## 安裝方式

### 1. 安裝 Python 套件

```bash
pip install opencv-python pyzbar Pillow pywin32 openpyxl xlwings
```

### 2. 初始化 pywin32（必要步驟）

安裝 pywin32 後需執行後置安裝腳本，否則 COM 介面無法正常運作：

```bash
python Scripts/pywin32_postinstall.py -install
```

> ⚠️ 若 Excel 以系統管理員執行，本程式也需以**相同權限**啟動，否則 COM 連線會失敗。

### 3. 確認 ZBar 動態函式庫（Windows）

`pyzbar` 在 Windows 上需要 ZBar DLL，可參考 [pyzbar 官方說明](https://github.com/NaturalHistoryMuseum/pyzbar) 確認安裝方式。

## 使用方法

### 基本流程

```bash
python Qr_excel_scanner.py
```

1. 程式啟動後自動開啟攝影機，並切換至**簡易模式**靠右顯示。
2. 在 Excel 中點選要填入的儲存格（游標位置即為填入目標）。
3. 將條碼 / QR Code 對準鏡頭，點擊「▶ 開始掃描」。
4. 掃描成功後，資料自動填入 Excel 游標所在儲存格。
5. 若開啟「自動下移」，游標會在延遲後自動移至下一列，可連續掃描。

### 介面說明

| 按鈕 / 設定 | 功能 |
|------------|------|
| ▶ 開始掃描 / ⏹ 停止掃描 | 切換掃描狀態 |
| ⊟ 簡易 / ⊞ 完整 | 切換介面模式 |
| 📌 置頂 | 視窗永遠顯示在最上層 |
| 填入模式 | 開啟後，掃描結果自動寫入 Office |
| 自動下移 | 填入後游標自動向下移一列 |
| 下移延遲 | 自動下移前的等待秒數（0.1 ~ 3.0 秒） |
| 冷卻時間 | 同一條碼再次掃入需等待的秒數 |
| 篩選模式 | 全部 / 僅 MAC / 僅 SN |
| 擷取模式 | 完整內容 / MAC 後 N 碼 |
| 交替填入 | MAC 與 SN 自動交替填入相鄰欄 |
| 數位放大 | 1× ~ 4×，鼠標拖拽調整放大中心 |
| COM 診斷 | 顯示 Office COM 連線狀態，協助排查問題 |

## 技術架構

```
Qr_excel_scanner.py
│
├── 攝影機擷取層（OpenCV）
│   ├── cv2.VideoCapture：鏡頭開啟 / 解析度設定
│   ├── 數位放大（ROI 裁切 + resize）
│   └── 自動曝光補償（亮度分析 + V channel 調整）
│
├── 條碼解碼層（pyzbar）
│   ├── 背景執行緒掃描（與 UI 執行緒分離）
│   ├── Code128 強化前處理（對比增強 + Otsu 二值化）
│   └── 結果節流（冷卻時間 + 次數上限）
│
├── Office COM 整合層（pywin32）
│   ├── 常駐 COM 執行緒（STA apartment 無洩漏設計）
│   ├── Excel：GetActiveObject → ActiveSheet.Cells(row, col).Value
│   ├── Word：Selection.TypeText()
│   └── COM 診斷工具（_diagnose_office_com）
│
├── 圖形介面層（Tkinter）
│   ├── 完整模式 / 簡易模式切換
│   ├── Toast 通知（子視窗，可拖動 / 釘選）
│   ├── 設定持久化（JSON，背景執行緒讀寫）
│   └── 視窗置頂 / 位置記憶
│
└── 設定管理
    └── qr_scanner_settings.json（同目錄自動產生）
```

### 核心技術細節

**COM 單一執行緒設計**
pywin32 的 COM 物件必須在同一 STA（Single-Threaded Apartment）執行緒操作，否則會造成 ROT（Running Object Table）殘留。本專案採用**單一常駐 COM 執行緒 + 請求佇列**架構，所有 COM 操作統一調度，徹底避免 apartment 洩漏問題。

**焦距學習機制**
每次成功掃碼時記錄當前焦距為「黃金焦距」，閒置超過設定時間後自動鎖定至黃金焦距值，有效防止自動對焦在無目標時漂移造成下次掃描失敗。

**前導零保護**
當掃描結果為純數字且以 `0` 開頭時，填入前自動將儲存格格式設定為文字（`NumberFormat = "@"`），確保前導零不被 Excel 截斷。

## 常見問題

**Q：找不到 Excel / COM 連線失敗？**
A：請確認 Excel 已開啟且有活頁簿。點擊「COM 診斷」按鈕查看詳細錯誤訊息。若 Excel 以管理員身分執行，本程式也需以管理員身分執行。

**Q：pyzbar import 失敗？**
A：Windows 上需要 ZBar DLL，請參考 pyzbar 文件確認 DLL 是否正確安裝於系統路徑。

**Q：攝影機無法開啟？**
A：嘗試調整「Camera Index」（0、1、2...），或確認攝影機未被其他程式佔用。

**Q：掃描率低 / 辨識困難？**
A：啟用「條碼強化模式」、增加數位放大倍率，或在光線較充足的環境使用。閒置鎖焦功能可保持焦距穩定。
